# ホスト

サイトと実稼動前の環境を整理し、での管理と個別レポートを容易にします。 [!DNL Adobe Target].

ホスト管理の主な目的は、非アクティブなコンテンツが誤って Web サイトに表示されるのを確実に防ぐことです。ホスト管理では、 [環境](/help/administrating-target/environments.md).

ホストは、 [!DNL Target] リクエストが作成されました。 Web サイトでは、通常、 `location.hostname` プロパティの値を指定し、 [!DNL Target] リクエスト。

デフォルトでは、 [!DNL Target] が作成できるホストを制限しない [!DNL Target] 要求と受信 [!DNL Target] 応答。 新しいホストがリクエストをおこなうと、自動的に機能します。 また、このプロセスにより、不明なドメインや予期できないドメインでのテストも可能になります。 このデフォルトの動作を上書きする場合は、どのホストを使用するかを制限す許可リストるまたブロックリストはオプションを設定できます [!DNL Target].

ホストを管理するには、 **[!UICONTROL 管理]** > **[!UICONTROL ホスト]**.

## ホストの認識 {#concept_0D4B43E23AA9408F8B28A57ED754BF65}

ホストを認識し、 [!UICONTROL ホスト] リストに含めるには、次の条件を満たす必要があります。

* 少なくとも 1 つ [!DNL Target] リクエストはホスト上に存在する必要があります
* ホスト上のページには、以下のものが含まれている必要があります。

   * 正確な at.js の参照
   * A [!DNL Target] リクエストまたは自動生成されたグローバル [!DNL Target] リクエスト

* ページ [!DNL Target] リクエストはブラウザーで表示する必要があります

ページが表示されると、ホストが [!UICONTROL ホスト] リストを作成して、環境で管理したり、アクティビティとテストをプレビューして開始したりできます。

>[!NOTE]
>
>この説明は、個人で使用する開発サーバーも対象となります。

ホストが[!UICONTROL ホスト]リストに追加された後、確実にホストが認識されるようにします。

1. クリック **[!UICONTROL 管理]** > **[!UICONTROL ホスト]**.
1. ホストが一覧に表示されていない場合は、ブラウザーを更新します。

   デフォルトでは、新しく認識されたホストは [!UICONTROL 実稼動] 環境。 この [!UICONTROL 実稼動] 環境は最も安全な環境です。非アクティブなアクティビティをこれらのホストから表示できないからです。

1. （条件付き） **[!UICONTROL 移動]** アイコン ( ![移動アイコン](/help/administrating-target/assets/icon-move.png) ) をクリックして、ホストを [!UICONTROL 開発], [!UICONTROL ステージング]、または他の環境。

>[!NOTE]
>
>この [!UICONTROL 実稼動] 環境は、名前を変更した場合でも削除できません。 この環境で、最終的なアクティブなアクティビティとテストを扱うことを前提としています。 デフォルトの環境では、非アクティブなキャンペーンの表示は許可されていません。
