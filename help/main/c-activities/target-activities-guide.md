# [!DNL Target] アクティビティタイプ

様々なアクティビティPDFについて説明するインタラクティブタイプのダウンロード [!DNL Adobe Target].

>[!NOTE]
>
>注意：最適なエクスペリエンスを実現するには、インタラクティブな [Adobe Target アクティビティガイド PDF](/help/main/assets/activities_guide_82817.pdf) をダウンロードしてください。他の人と共有することもできます。
>
>この記事には、 [[!UICONTROL Recommendations] アクティビティ](/help/main/c-recommendations/recommendations.md). ただし、内にレコメンデーションを組み込むことはできます [!UICONTROL A/B テスト], [!UICONTROL 自動配分], [!UICONTROL 自動ターゲット]、および [!UICONTROL エクスペリエンスのターゲット設定] (XT) アクティビティ 詳細については、「[ オファーとしての Recommendations](/help/main/c-recommendations/recommendations-as-an-offer.md)」をご覧ください。この機能を使用するには、[Target Premium ライセンス](/help/main/c-intro/intro.md#premium)が必要です。

## 各アクティビティの機能  {#section_4ECAACC68723402EB3649033190E1BBC}

| アクティビティタイプ | 詳細 |
|--- |--- |
| 手動 [!UICONTROL A/B テスト]<P>![アイコン](/help/main/c-activities/assets/icon_ab.png) | 複数のエクスペリエンスを比較し、事前に指定したテスト期間を通して、コンバージョンを最も増やしたエクスペリエンスを見極めます。<P>詳しくは、[A/B テスト](/help/main/c-activities/t-test-ab/test-ab.md)を参照してください。 |
| [!UICONTROL 自動配分]<P>![アイコン自動配分](/help/main/c-activities/assets/icon_auto_allocate.png) | 2 つ以上のエクスペリエンスの中から勝者を特定した後、トラフィックを勝者エクスペリエンスにリダイレクトし、テストと学習に合わせてコンバージョンを増やします。<P>詳しくは、 [自動配分](/help/main/c-activities/automated-traffic-allocation/automated-traffic-allocation.md)を参照してください。 |
| [!UICONTROL 自動ターゲット]<P>![アイコン AT](/help/main/c-activities/assets/icon_auto_target.png) | 高度な機械学習を使用してコンテンツをパーソナライズし、コンバージョンを促進します。その際は、マーケターが定義した高パフォーマンスのエクスペリエンスを複数特定したうえで、個々の顧客プロファイルや同様の訪問者の過去の行動を基にして、各訪問者に詳細にカスタマイズしたエクスペリエンスを配信します。<P>詳しくは、[パーソナライズされたエクスペリエンスの自動ターゲット](/help/main/c-activities/auto-target/auto-target-to-optimize.md)を参照してください。 |
| [!UICONTROL Automated Personalization]（AP）<P>![アイコン AP](/help/main/c-activities/assets/icon_ap.png) | 高度な機械学習を使用して特定のオファーやメッセージを組み合わせて、個々の顧客プロファイルに合わせて各訪問者に様々なオファーのバリエーションを表示しながら、コンテンツをパーソナライズしてコンバージョンを促進していきます。<P>詳しくは、[Automated Personalization](/help/main/c-activities/t-automated-personalization/automated-personalization.md) を参照してください。 |
| [!UICONTROL 多変量分析テスト] （MVT）<P>![アイコン MVT](/help/main/c-activities/assets/icon_mvt.png) | ページ上の要素間でオファーの組み合わせを比較し、特定のオーディエンスに対して最も効果が高いオファーの組み合わせを確認します。 事前に指定したテスト期間内で、コンバージョンを最も増やすことができたページ要素を見極めることもできます。<P>詳しくは、[多変量分析テスト](/help/main/c-activities/c-multivariate-testing/multivariate-testing.md)を参照してください。 |
| エクスペリエンスのターゲット設定（XT）<P>![アイコン XT](/help/main/c-activities/assets/icon_xt.png) | マーケターが定義した一連のルールや条件を基にして、特定のオーディエンスにコンテンツを配信します。<P>詳しくは、[エクスペリエンスのターゲット設定](/help/main/c-activities/t-experience-target/experience-target.md)を参照してください。 |

## このタイプのアクティビティを使用する理由 {#section_46A70DD7CE3448749E635DDF5EAFC131}

| アクティビティタイプ | 理由 |
|--- |--- |
| 手動 [!UICONTROL A/B テスト] | 高度に制御されたテストで、トラフィックを測定し、ルールではなく割合で分割します。テストデータを分析してオーディエンスに関するインサイトを抽出し、どのエクスペリエンスが最も効果的か見極めることができます。 |
| [!UICONTROL 自動配分] | 勝者エクスペリエンスを特定し、トラフィック配分を調整して、可能な限り早く訪問者に勝者エクスペリエンスを提供する方法を、より迅速かつ高いコンバージョンの可能性をサポートします。 |
| [!UICONTROL 自動ターゲット] | 複数のエクスペリエンスの中から勝者を見つけ出し、特定の訪問者に最適なエクスペリエンスを配信します。特定の期間、特定のエクスペリエンスを対象にアルゴリズムが訪問者のコンバージョンの傾向を推測するので、訪問者の興味の変化に合わせてターゲット設定が調節されていきます。 |
| [!UICONTROL Automated Personalization]（AP） | （単一のページまたは複数のページの要素を対象に、作成または事前定義された）一連のオファーをパーソナライズし、特定の訪問者を引き付けるうえで最も効果的な組み合わせを配信します。 |
| [!UICONTROL Multivariate Testing] (MVT) | 複数の要素に複数のオファーを表示し、結果として得られる一意のエクスペリエンスを特定の目標に対して同時にテストする方法で、どの要素のバリエーションが最も効果が高いかを判断できます。 MVT アクティビティでは、訪問者のインタラクションに最もプラスまたはマイナスの影響を与えた要素を明らかにすることもできます。 |
| [!UICONTROL エクスペリエンスのターゲット設定]（XT） | 定義された一連の割り当てルールに基づいて、特定のオーディエンスに対する特定のコンテンツをターゲット設定する方法。 |

## どのようなマーケターがアクティビティのタイプを使用すべきですか？ {#section_A843D663D3E543FFB1A594266B560395}

| アクティビティタイプ | マーケティング担当者 |
|--- |--- |
| 手動 [!UICONTROL A/B テスト] | 統計に関する知識が豊富。<P>結果を分析する際に、テスト期間が終わるまで待つことができる。 |
| [!UICONTROL 自動配分] | 時間に余裕がない。<P>最適なエクスペリエンスを迅速に特定し、配信する必要がある。<P>テストの実施途中に結果を「のぞき見たい」。 |
| [!UICONTROL 自動ターゲット] | 効果的なエクスペリエンスが複数ある。<P>動的に変化するプロファイルに合わせて、特定の訪問者に最適なタイミングでエクスペリエンスを配信したい。 |
| [!UICONTROL Automated Personalization]（AP） | 1 つ以上のオファーがある。<P>様々な独自のプロファイルや行動を考慮し、特定の訪問者に合わせて最適なパーソナライズされたエクスペリエンスを創出できるオファーの組み合わせを作成したい。 |
| [!UICONTROL Multivariate Testing] (MVT) | 統計に関する知識が豊富。<P>1 つ以上のオファーがある。<P>ページ要素のインタラクションに関連するコンバージョンの傾向を分析したい。 |
| [!UICONTROL エクスペリエンスのターゲット設定]（XT） | 特定のオーディエンスに特定のエクスペリエンスまたはコンテンツの一部を配信する必要がある。 |

## 統計面の詳細 {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| アクティビティタイプ | 詳細 |
|--- |--- |
| 手動 [!UICONTROL A/B テスト] | テストでは、各挑戦者のエクスペリエンスをコントロールエクスペリエンスと比較し、すべてのエクスペリエンスのパフォーマンスをランク付けし、コントロールと比較した場合に勝者エクスペリエンスと敗者エクスペリエンスの両方を識別します。 |
| [!UICONTROL 自動配分] | このテストでは、すぐに真の勝者が統計的に保証され、コンバージョンに至る可能性が高いオーディエンスに、その勝者のエクスペリエンスがより多く配信されるようになります。 |
| [!UICONTROL 自動ターゲット] | 最適化メカニズムでは、時間の経過と共に上昇率の増減を示し、どのエクスペリエンスにどの訪問者を配信するかを事前に決定することで、各エクスペリエンスの関連オーディエンスを特定します。 最適化メカニズムは、コンバージョン、セグメント、パラメーターおよびプロファイルスクリプトによって通知されます。 このメカニズムは、上昇率とコンバージョン率を高めるために使用するアルゴリズムを自動的に選択します。 |
| [!UICONTROL Automated Personalization]（AP） | 最適化メカニズムによって、新規訪問者の行動や、類似する訪問者の過去の行動などに基づき、どのエクスペリエンスをどの訪問者に配信するかが常に調節され、オファーのパフォーマンスが同時並行で配信されているコントロール母集団と比較されます。 |
| [!UICONTROL Multivariate Testing] (MVT) | 特定の要素がコンバージョンに与える相対的な影響を把握するのに役立ちます。 |
| [!UICONTROL エクスペリエンスのターゲット設定]（XT） | この手法では、特定のエクスペリエンスか、コンテンツの特定部分のターゲットを特定のオーディエンスに設定するルールを定義します。お客様は、エクスペリエンスレベルで更新をおこなうことができます。 |

## 利点と注意点 {#section_56C46ABEF7B945DDA0C1E6D714377123}

| アクティビティタイプ | 利点 | 注意点 |
|--- |--- |--- |
| 手動 [!UICONTROL A/B テスト] | A/B 休憩により、どのエクスペリエンスが最も効果が高いかだけでなく、各エクスペリエンスのパフォーマンスを完全に理解できます。 | 内、 [!UICONTROL A/B テスト]サンプルサイズに達する前にテスト結果を見ると、不正確な結果に依存するリスクがあります（以前の「のぞき見することはできません」）。<P>取り消し [!UICONTROL 自動配分]A/B テストでは、一部のエクスペリエンスが他よりもパフォーマンスが優れていることを認識した後でも、トラフィックの配分方法は変わりません。<P>のベストプラクティスに関する情報 [!UICONTROL A/B テスト] アクティビティについては、 [A/B テストを実行すべき期間](/help/main/c-activities/t-test-ab/sample-size-determination.md) および [一般的な A/B テストの落とし穴 10 件と回避方法](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md). |
| [!UICONTROL 自動配分] | [!UICONTROL 自動配分では、手動の A/B テストよりも全体的なコンバージョン率が高いので、通常の A/B テストのコストを下げることができます。]コンバージョン率は、 [!UICONTROL 自動配分] が最もパフォーマンスの高いエクスペリエンスにより多くのトラフィックをプッシュします。つまり、テスト期間が終わる前に、その勝者エクスペリエンスのメリットを実現できます（プレビューできます）。 | [!UICONTROL この手法では、勝者は特定されますが、敗者は区別されずに処理されます。]各エクスペリエンスのパフォーマンスを把握しておく必要がある場合は、A/B テストをお勧めします。<P>The [!UICONTROL 自動配分] 機能は、詳細指標設定の 1 つ（「カウントを増分、アクティビティでユーザーを保持」）でのみ使用できます。 リピートコンバージョンをカウントしない場合は、代わりに A/B テストを使用する必要があります。 |
| [!UICONTROL 自動ターゲット] | を使用 [!UICONTROL 自動ターゲット]の場合、機械学習は、複数ページのエクスペリエンスを含む、あらゆる種類のエクスペリエンスに適用されます。 An [!UICONTROL 自動ターゲット] 「 」アクティビティでは、 [!UICONTROL Automated Personalization] 使い慣れた A/B テストワークフローを使用して | を使用 [!UICONTROL 自動ターゲット]を使用する場合、頻繁または頻繁にオファーのコンテンツを変更する場合は、アルゴリズムが各変更の後に、アルゴリズムが学習した内容を活用し、適切な訪問者にコンテンツを配信するのに十分な時間が必要です。 |
| [!UICONTROL Automated Personalization]（AP） | を使用 [!UICONTROL Automated Personalization]を使用すると、すべてのオファーを 1 か所に収集でき、最適なオファーの組み合わせがアルゴリズムによって決定されます。 個々のエクスペリエンスを指定したり作成したりする必要はありません。[!UICONTROL Automated Personalization] はと同じ機械学習アルゴリズムを使用します [!UICONTROL 自動ターゲット]. | 複数のオファーを組み合わせる場合は、組み合わせ爆発が発生するので、膨大な量のトラフィックが必要になります。The [!UICONTROL Automated Personalization] アルゴリズムは、多くの要因を考慮するので、必要なトラフィック量は最も多くなります。<P>[!UICONTROL Automated Personalization] では、レポートを使用できません [Analytics for Target](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T)。 |
| [!UICONTROL Multivariate Testing] (MVT) | を使用 [!UICONTROL Multivariate Testing]を使用すると、複数の要素を同時にテストできます。 | A [!UICONTROL 多変量分析テスト] は時間がかかり、再生中の複数の変数が原因で、必ずしも信頼性のある勝者エクスペリエンスを生み出すわけではありません。<P>多くの場合、テストの完了に必要なトラフィック量を確保することが課題になります。すべての [!UICONTROL 多変量分析テスト] 実験は全因子なので、一度に変化する要素が多すぎると、テストが必要な多くの組み合わせをすばやく追加できます。<P>トラフィックがかなり多いサイトでも、25 を超える組み合わせを含むテストは、妥当な時間内での完了が困難になる場合があります。 |
| [!UICONTROL エクスペリエンスのターゲット設定]（XT） | を使用 [!UICONTROL エクスペリエンスのターゲット設定]を使用すると、任意のアクティビティの結果から推定されるインサイトにすばやく対処できます。<P>例えば、挑戦者が対照を上回らなかった A/B テストを実行した場合に、その結果、訪問者の特定のセグメントが対照を使用した場合と比べて、対照を使用した場合のコンバージョンが訪問者の特定のセグメントの 4 倍増えた場合、 [!UICONTROL エクスペリエンスのターゲット設定] 特定のセグメントに挑戦者エクスペリエンスを誘導する。 | [!UICONTROL エクスペリエンスのターゲット設定では、複数のオーディエンスを対象に、1 つのエクスペリエンスの割合を設定することはできません。] |
