# [!DNL Target]リリースノート（プレリリース）

## [!DNL Target Standard/Premium] 25.8.2 （2025 年 8 月 14 日（PT））

このリリースには、次の修正および更新が含まれています。

**[!UICONTROL Activities]**

+++詳細を表示
* **更新された [!DNL Target] UI でのアクティビティの読み込みの問題を修正**：更新された [!DNL Target] UI で、編集を試みたときに特定のアクティビティを読み込めない問題を修正しました。 この問題により、お客様のユーザーが読み込み画面が無限に表示されていました。 この問題は複数のアクティビティに影響を与え、顧客間で発生が一貫していないことが報告されていました。 この修正により、影響を受けるアクティビティが適切に読み込まれるようになり、シームレスな編集が可能になり、アクティビティワークフローの中断を減らすことができます。 （TGT-53209）
* **`optionLocalId` 検証による、アクティビティ作成プロセスの保存エラーを修正しました**：バックエンドの検証エラーが原因で、顧客が変更を保存できなかったアクティビティ作成プロセスの問題を修正しました。`OptionLocalIdReferentialIntegrity.ABActivity - Invalid optionLocalIds:` この問題は、アクティビティ内の特定の要素を変更する際に発生し、保存操作に失敗したことによって発生しました。 この修正により、`optionLocalId` 参照が正しく検証され、このエラーが発生しなくてもアクティビティを保存できるようになります。 （TGT-53293）
* **ページ切り替え時の無効なオプション参照が原因で、アクティビティ作成プロセスでクラッシュが発生する修正**：編集中にページを 3 回切り替えると、UI がクラッシュするアクティビティ作成プロセスの問題を修正しました。 無効なオプション参照が原因でクラッシュがトリガーされ、「localId &#39;7&#39;のオプションが見つかりません」などのコンソールエラーが発生しました。 この更新により、システム障害や中断を発生させずに、ページを切り替えて変更を適用できるようになりました。 （TGT-53295）
* **エクスペリエンスの編集時に無効なユーザー入力が原因で発生する、アクティビティ作成プロセスの保存エラーを修正しました**：無効なユーザー入力エラーが原因で、顧客がアクティビティに対する変更を保存できなかったアクティビティ作成プロセスの問題を修正しました。 エラーが発生したのは、更新された UI でエクスペリエンスを変更する際で、更新をコミットできませんでした。 アクティビティを正常に保存できるようになり、顧客は中断することなく編集および公開できます。 （TGT-53267）
* **アクティビティ作成プロセスの読み込み問題が修正され、更新された UI での編集がブロックされました**：読み込み画面が継続的に表示されるので、顧客が更新された UI のアクティビティを編集できなかったアクティビティ作成プロセスの問題を修正しました。 読み込みエラーに遭遇することなく、アクティビティを開いたり変更したりできるようになりました。 （TGT-53415）

+++

**エクスペリエンスフラグメント（XF）**

+++詳細を表示
* **activity-create プロセスで、AEMで書き出されたフラグメントを意図せずにHTMLで編集できていた問題を修正しました**:activity-create プロセスで、お客様が、[!DNL Experience Fragments] 内の [!DNL Adobe Experience Manager] （AEM）から書き出された [!DNL Target] （XF）のHTMLを編集できた問題を修正しました。 XF はAEMから公開された後も編集用にロックされたままなので、この動作は意図されたものではありません。 この修正により、「HTMLを編集」オプションをAEMで書き出したフラグメントで使用できなくなり、コンテンツの整合性と予想されるガバナンスが維持されます。 （TGT-53286）

+++

**QA モード**

+++詳細を表示
* **アクティビティ作成プロセスで、URL の先頭のスペースが原因で QA リンクが壊れる問題を修正しました**：アクティビティ作成プロセスで、保存時にアクティビティ URL の先頭のスペースが正しく削除されなかった問題を修正しました。 これにより、無効な QA リンクが発生し、バックエンドの形式が正しくなくなりました。 更新後、URL はクリーンに保存されるようになり、リンク切れを防ぎ、顧客に対する QA ワークフローの信頼性を向上させることができます。 （TGT-53427）

+++

**[!UICONTROL Recommendations]**

+++詳細を表示
* **カタログ検索 UI で詳細検索で提案を提供できなかった問題を修正しました**：新しい [!UICONTROL Catalog Search] UI で、[!UICONTROL Advanced Search] 機能で提案を提供できなかった問題を修正しました。 ユーザーは正確なスペルの正確な値を入力する必要があり、検索が煩雑でエラーが発生しやすくなっていました。 この修正により、[!UICONTROL Advanced Search] はユーザーのタイプに応じて関連性の高い提案を提供するようになり、操作性が向上し、製品を見つける際の摩擦が軽減されます。 （TGT-52008）
* **複数の UI とフィルターの問題を解決して、応答性とエンティティのインタラクションを向上**：小画面デバイスでの条件の詳細の応答性が悪い、環境フィルターで「すべてのホストグループ」を選択した際に結果が得られない、名前のないエンティティを操作できないなど、いくつかの問題を解決しました。 これらの修正により、画面サイズをまたぐユーザビリティが向上し、正確なフィルタリングが保証され、すべての製品エンティティとの完全なやり取りが可能になり、ユーザーの全体的なエクスペリエンスが向上します。 （TGT-52992）
* **アクティビティの作成中に Recommendations の詳細ビューに製品 ID が表示されない問題を修正**:[!DNL Recommendations] アクティビティの作成プロセスで、製品 ID が製品の詳細画面に表示されず、ワークフロー中に製品 ID をコピーまたは参照するのが難しかった問題を修正しました。 製品 ID が詳細ビューに明確に表示されるようになり、可視性が向上し、顧客のより効率的な製品管理がサポートされるようになりました。 （TGT-51964）
* **レコメンデーションビューで製品メッセージの表示に失敗した、アクティビティ作成プロセスの問題を修正しました**：メッセージが存在しても、レコメンデーションビューの [!UICONTROL Message] 列に製品データが表示されない [!DNL Recommendations] クティビティ作成プロセスの問題を修正しました。 顧客は、コンテンツを一時的に表示するために手動で列を削除および再追加する必要があり、多くの場合、スクロールや検索の際に再び表示されなくなります。 この更新により、製品メッセージの一貫性のある表示が復元され、カタログナビゲーションとレビューワークフローが改善されました。 （TGT-52777）
* **アクティビティ作成プロセスで、「すべてのホストグループ」を選択しても Recommendations ビューで結果が返されない問題を修正しました**：アクティビティ作成プロセスで、[!DNL Recommendations] ビューで「すべてのホストグループ」環境を選択しても結果が返されない問題を修正しました。 お客様は、すべてのホストグループの製品データを期待どおりに表示できるようになり、アクティビティの設定時の可視性とフィルタリングの精度が向上しました。 （TGT-53006）
* **保存後にフロントプロモーションの切り替えが保持されない、アクティビティ作成プロセスの問題を修正しました**：保存後、アクティビティ設定でフロントプロモーションの切り替えが保持されない、アクティビティ作成プロセスの問題を修正しました。 フロントプロモーションを削除しようとしたお客様は、アクティビティを再訪問するとトグルが再び有効になり、適切なカスタマイズが妨げられました。 この更新により、変更を確実に保存でき、お客様はプロモーション設定を完全に制御できます。 （TGT-53215）

+++

**Visual Experience Composer（VEC）**

+++詳細を表示
* **アクティビティ作成プロセスにおけるアクティビティの読み込みと [!UICONTROL Cancel] ボタンの問題を修正**：アクティビティ作成プロセスで、特定のアクティビティを読み込むことができず、顧客が変更にアクセスできない問題を修正しました。 また、「[!UICONTROL Cancel]」ボタンが応答しなくなり、顧客は読み込みプロセスを停止したり、編集ビューを終了したりできませんでした。 この修正により、アクティビティが確実に読み込まれ、「[!UICONTROL Cancel]」ボタンが期待どおりに機能するようになり、Visual Experience Composer の全体的な安定性とユーザーエクスペリエンスが向上します。 （VEC）（TGT-53218）
* **更新された VEC UI でのエクスペリエンス切り替えの問題を修正し、編集をブロックし、[!UICONTROL Cancel] ボタンを無効にしました**：更新された VEC UI で、エクスペリエンスのターゲット設定（XT）アクティビティでエクスペリエンスを切り替えると変更が読み込まれなかった問題を修正しました。 顧客が最初に入力したエクスペリエンス以外のエクスペリエンスを編集できず、「[!UICONTROL Cancel]」ボタンが見つからないか、応答しませんでした。 この修正により、すべてのエクスペリエンスで変更が正しく読み込まれ、ヘルパー拡張機能がなくても [!UICONTROL Cancel] ボタンが確実に機能するようになり、編集ワークフローが改善され、フラストレーションが軽減されます。 （TGT-53256）
* **アクティビティ作成プロセスで複数のエクスペリエンスを切り替える際の白い画面の問題を修正しました**：複数のエクスペリエンスを切り替えると白い画面が表示される問題を修正しました。 また、アクティビティ作成プロセスで、1 つのアクティビティ内の複数のエクスペリエンスを変更しようとすると、白い画面が表示される問題を修正しました。 この問題は、2 つのエクスペリエンスに変更を適用してから 3 つ目のエクスペリエンスを選択し、それ以上の編集を行わなかった場合に発生していました。 この更新により、エクスペリエンス間のスムーズなトランジションが保証され、お客様は中断することなく変更できます。 （TGT-53266）
* **編集セッションをまたいでカスタムコードの変更が確実に保存されない VEC の問題を修正しました**:Visual Experience Composer （VEC）で行われたカスタムコードの変更が 1 回の試行で確実に保存されない問題を修正しました。 お客様から、「[!UICONTROL Edit Content]」ボタンと「final [!UICONTROL Save]」ボタンの両方を使用した後でも、スタイル設定の更新やHTMLの編集などの変更が web サイトや QA URL で断続的に欠落していることが報告されました。 このリグレッションは解決され、すべてのカスタムコードの変更が編集セッション間で期待どおりに維持されるようになりました。** （TGT-53418）
