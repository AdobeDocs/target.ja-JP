# [!DNL Target]リリースノート（プレリリース）

この記事には、SDK、API、JavaScript ライブラリなど、次回の [!DNL Adobe Target] リリースのプレリリース情報が含まれています。

**最終更新日：2025 年 7 月 24 日**

>[!NOTE]
>
>* リリース日、機能などの情報は、予告なく変更されることがあります。
>
>* 最新のリリースに関する情報を確認するには、[Target リリースノート ](release-notes.md) を参照してください。
>
>* 括弧内の問題番号は [!DNL Adobe] 内部で使用されます。

## [!DNL Target Standard/Premium] 25.8.2 （2025 年 8 月 13 日（PT））

このリリースには、次の修正および更新が含まれています。

**[!UICONTROL Activities]**

+++詳細を表示
* 更新された [!DNL Target] UI で、編集しようとすると特定のアクティビティが読み込まれない問題を修正しました。 この問題により、お客様のユーザーが読み込み画面が無限に表示されていました。 この問題は複数のアクティビティに影響を与え、顧客間で発生が一貫していないことが報告されていました。 この修正により、影響を受けるアクティビティが適切に読み込まれるようになり、シームレスな編集が可能になり、アクティビティワークフローの中断を減らすことができます。 （TGT-53209）
* アクティビティ作成プロセスで、バックエンドの検証エラーが原因で顧客が変更を保存できない問題を修正しました。`OptionLocalIdReferentialIntegrity.ABActivity - Invalid optionLocalIds:` この問題は、アクティビティ内の特定の要素を変更する際に発生し、保存操作に失敗していました。 この修正により、`optionLocalId` 参照が正しく検証され、このエラーが発生しなくてもアクティビティを保存できるようになります。 （TGT-53293）
* アクティビティ作成プロセスで、編集中にページを 3 回切り替えると UI がクラッシュする問題を修正しました。 無効なオプション参照が原因でクラッシュがトリガーされ、「localId &#39;7&#39;のオプションが見つかりません」などのコンソールエラーが発生しました。 この更新により、システム障害や中断を発生させずに、ページを切り替えて変更を適用できるようになりました。 （TGT-53295）
* アクティビティ作成プロセスで、無効なユーザー入力エラーが原因で顧客がアクティビティに対する変更を保存できなかった問題を修正しました。 エラーが発生したのは、更新された UI でエクスペリエンスを変更する際で、更新をコミットできませんでした。 アクティビティを正常に保存できるようになり、顧客は中断することなく編集および公開できます。 （TGT-53267）
* アクティビティ作成プロセスで、読み込み画面が連続していたために、顧客が更新された UI のアクティビティを編集できなかった問題を修正しました。 読み込みエラーに遭遇することなく、アクティビティを開いたり変更したりできるようになりました。 （TGT-53415）

+++

**エクスペリエンスフラグメント（XF）**

+++詳細を表示
* アクティビティ作成プロセスで、お客様が [!DNL Experience Fragments] （AEM）から書き出されたHTML of [!DNL Adobe Experience Manager] （XF）を [!DNL Target] 内で編集できた問題を修正しました。 XF はAEMから公開された後も編集用にロックされたままなので、この動作は意図されたものではありません。 この修正により、「HTMLを編集」オプションをAEMで書き出したフラグメントで使用できなくなり、コンテンツの整合性と予想されるガバナンスが維持されます。 （TGT-53286）

+++

**QA モード**

+++詳細を表示
* アクティビティ作成プロセスで、保存時にアクティビティ URL の先頭のスペースが正しく削除されなかった問題を修正しました。 これにより、無効な QA リンクが発生し、バックエンドの形式が正しくなくなりました。 更新後、URL はクリーンに保存されるようになり、リンク切れを防ぎ、顧客に対する QA ワークフローの信頼性を向上させることができます。 （TGT-53427）

+++

**[!UICONTROL Recommendations]**

+++詳細を表示
* 新しい [!UICONTROL Catalog Search] UI で、[!UICONTROL Advanced Search] 機能が提案を提供できなかった問題を修正しました。 ユーザーは正確なスペルの正確な値を入力する必要があり、検索が煩雑でエラーが発生しやすくなっていました。 この修正により、[!UICONTROL Advanced Search] はユーザーのタイプに応じて関連性の高い提案を提供するようになり、操作性が向上し、製品を見つける際の摩擦が軽減されます。 （TGT-52008）
* 小画面のデバイスで条件の詳細の応答性が悪い、[!UICONTROL Environment] フィルターで「すべてのホストグループ」を選択しても結果が得られない、名前のないエンティティを操作できないなど、いくつかの問題を解決しました。 これらの修正により、画面サイズをまたぐユーザビリティが向上し、正確なフィルタリングが保証され、すべての製品エンティティとの完全なやり取りが可能になり、ユーザーの全体的なエクスペリエンスが向上します。 （TGT-52992）
* [!DNL Recommendations] アクティビティ作成プロセスで、製品の詳細画面に製品 ID が表示されず、ワークフロー中に製品 ID をコピーまたは参照するのが難しい問題を修正しました。 製品 ID が詳細ビューに明確に表示されるようになり、可視性が向上し、顧客のより効率的な製品管理がサポートされるようになりました。 （TGT-51964）
* アクティビティ作成プロセスで、メッセージが存在する場合でも、[!UICONTROL Message] ビューの [!DNL Recommendations] 列に製品データが表示されない問題を修正しました。 顧客は、コンテンツを一時的に表示するために手動で列を削除および再追加する必要があり、多くの場合、スクロールや検索の際に再び表示されなくなります。 この更新により、製品メッセージの一貫性のある表示が復元され、カタログナビゲーションとレビューワークフローが改善されました。 （TGT-52777）
* アクティビティ作成プロセスで、[!DNL Recommendations] ビューで「すべてのホストグループ」環境を選択しても結果が返されない問題を修正しました。 お客様は、すべてのホストグループの製品データを期待どおりに表示できるようになり、アクティビティの設定時の可視性とフィルタリングの精度が向上しました。 （TGT-53006）
* アクティビティ作成プロセスで、保存した後にアクティビティ設定でフロントのプロモーション切り替えスイッチを無効にしても保持されない問題を修正しました。 フロントプロモーションを削除しようとしたお客様は、アクティビティを再訪問するとトグルが再び有効になり、適切なカスタマイズが妨げられました。 この更新により、変更を確実に保存でき、お客様はプロモーション設定を完全に制御できます。 （TGT-53215）

+++

**Visual Experience Composer（VEC）**

+++詳細を表示
* アクティビティ作成プロセスで、特定のアクティビティの読み込みに失敗し、顧客が変更にアクセスできなくなる問題を修正しました。 また、「[!UICONTROL Cancel]」ボタンが応答しなくなり、顧客は読み込みプロセスを停止したり、編集ビューを終了したりできませんでした。 この修正により、アクティビティが確実に読み込まれ、「[!UICONTROL Cancel]」ボタンが期待どおりに機能するようになり、Visual Experience Composer （VEC）での全体的な安定性とユーザーエクスペリエンスが向上します（TGT-53218）
* 更新された VEC UI で、エクスペリエンスのターゲット設定（XT）アクティビティでエクスペリエンスを切り替えると変更が読み込まれない問題を修正しました。 顧客が最初に入力したエクスペリエンス以外のエクスペリエンスを編集できず、「[!UICONTROL Cancel]」ボタンが見つからないか、応答しませんでした。 この修正により、すべてのエクスペリエンスで変更が正しく読み込まれ、ヘルパー拡張機能がなくても [!UICONTROL Cancel] ボタンが確実に機能するようになり、編集ワークフローが改善され、フラストレーションが軽減されます。 （TGT-53256）
* 複数のエクスペリエンスを切り替えると白い画面が表示される問題を修正しました。 また、アクティビティ作成プロセスで、1 つのアクティビティ内の複数のエクスペリエンスを変更しようとすると、白い画面が表示される問題を修正しました。 この問題は、2 つのエクスペリエンスに変更を適用してから 3 つ目のエクスペリエンスを選択し、それ以上の編集を行わなかった場合に発生していました。 この更新により、エクスペリエンス間のスムーズなトランジションが保証され、お客様は中断することなく変更できます。 （TGT-53266）
* Visual Experience Composer （VEC）で行われたカスタムコードの変更が、1 回の試行で確実に保存されない問題を修正しました。 お客様から、「[!UICONTROL Edit Content]」ボタンと「final [!UICONTROL Save]」ボタンの両方を使用した後でも、スタイル設定の更新やHTMLの編集などの変更が web サイトや QA URL で断続的に欠落していることが報告されました。 このリグレッションは解決され、すべてのカスタムコードの変更が編集セッション間で期待どおりに維持されるようになりました。 （TGT-53418）
