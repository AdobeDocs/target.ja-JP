## [!DNL Target Standard/Premium] 25.8.4 （2025 年 8 月 28 日（PT））

このリリースには、次の更新および修正が含まれています。

**[!UICONTROL Activities]**

+++詳細を表示
* **スケジュールされたアクティビティの作成プロセス中に、更新された UI でスケジュールされたアクティビティの「[!UICONTROL Live]」ステータスが正しく表示されない問題を修正しました**：今後のアクティベーション日を持つアクティビティの「[!UICONTROL Scheduled]」ステータスが正しく表示されるようになりました。 （TGT-53187）
* **ページが更新されるまで、スケジュールされたアクティビティに誤って「[!UICONTROL Live]」ステータスが表示される**：顧客は、アクティビティが将来の日時に運用開始されるようにスケジュールすると、最初は [!UICONTROL Live] ではなく [!UICONTROL Scheduled] として表示されることを報告しました。 これは、確認メッセージが正しいスケジューリング動作を示しているにもかかわらず、混乱を引き起こしました。 ページを更新しなくても、保存直後のアクティビティのステータスが [!UICONTROL Scheduled] を正確に反映するようになりました。 （TGT-52937）
* **重複したエクスペリエンスへの変更は、元のエクスペリエンスに意図せず影響を与えました**：アクティビティ内のエクスペリエンスを複製し、新しいオーディエンスを割り当てると、複製されたエクスペリエンスに加えられた変更（デザインや条件など）が元のエクスペリエンスにも反映されることを顧客が報告しました。 これにより、独立したバリエーションを作成できなくなり、実稼動ワークフローに影響を与えました。 複製されたエクスペリエンスは、元のバージョンに影響を与えずに個別に編集できるようになりました。 （TGT-53361）
* **`InvalidProperty.Json` エラーが原因で、お客様がアクティビティを保存できない問題を修正しました**：以前は、変更を加えないでアクティビティを保存しようとすると、「ユーザー入力が無効です」というエラーが発生していました。 エラーは、JSON ペイロードに認識できないプロパティ名「content」が原因で発生しました。 この問題は解決され、変更を加えない場合でも、更新された UI でアクティビティを正常に保存できるようになりました。 （TGT-53513）

+++詳細を表示
* **A4T アクティビティを作成する際にレポートスイート名を入力できません**：更新された UI でレポートソースとして [!UICONTROL Report Suite] を選択した場合、お客様は [!UICONTROL Analytics] ドロップダウンに入力できませんでした。  この問題により、特にリストが大きい組織では、特定のレポートスイートを見つけるのが難しくなりました。 ドロップダウンで名前による入力と検索がサポートされるようになり、アクティビティ作成プロセスの効率が向上しました。 （TGT-53345）

**[!UICONTROL Audiences]**

+++詳細を表示
* **期限切れの「時間枠」オーディエンスにより、削除後もアクティビティの保存がブロックされていました**：以前は、期限切れの「時間枠」オーディエンスが含まれていた場合、そのオーディエンスが削除された後も、アクティビティを保存できませんでした。 この問題は、アクティビティのみのオーディエンスに対する検証が長引き、それがトリガーエラーを続けたことが原因です。 期限切れのオーディエンスが削除されると、アクティビティを期待どおりに保存できるようになりました。 （TGT-53517）
* **アクティビティを保存すると、追加のページと複数のオーディエンスが表示されなくなりました**：以前は、アクティビティ作成プロセスで [!UICONTROL A/B Test] アクティビティを作成する顧客の場合、保存後に、設定済みの追加のページと複数のオーディエンスが失われていました。 この動作は予期せず、セットアップ中に混乱が発生しました。 これらの設定は、アクティビティを保存した後も正しく保持されるようになりました。 （TGT-52357）

**決定オファー**

+++詳細を表示
* **更新された VEC で決定オファーを編集したり、特定のページ要素を選択したりできない**：更新された UI で複数の問題が発生し、既存のアクティビティを更新したり、新しいアクティビティを作成したりできなくなりました。

   * 決定オファーがHTML オファーとして誤って表示され、編集ができませんでした。
   * VEC で特定のページ要素を選択できませんでした。
   * 編集中に加えた変更は保存されませんでした。
   * 特定の地域の URL が VEC に正しく読み込まれず、手動の cookie 調整が必要になる。

  顧客は、決定オファーを編集し、ページ要素を正確に選択し、期待どおりに変更を保存できるようになりました。 地域ページも VEC に正しく読み込まれます。 （TGT-53425）

* **保存後、オファー決定セレクターを変更できず、予期せず変更することができませんでした**：更新された UI でオファー決定を適用する際に、セレクターをデフォルト値から変更できなかったことが報告されました。 セレクター（`#tf-cq-hr or body` など）を変更しようとしても無視され、アクティビティを保存した後、セレクターは `#cdq_element_0` のような汎用値に置き換えられました。 この問題により、オファーが Visual Experience Composer （VEC）から消え、それ以上の編集がブロックされていました。 顧客はオファーの決定セレクターを意図したとおりに変更できるようになり、保存済みのセレクターは予期しない変更を加えずに正しく保持されます。 （TGT-53433）
* **保存後、アクティビティからオファー決定が消えた**：更新された UI のページ要素に適用されたオファー決定が、アクティビティを保存すると表示されなくなったと顧客から報告されました。 場合によっては、セレクターが予期せず変更され、再編集時にオファーが VEC でレンダリングに失敗しました。 オファーの決定を保存後も正しく保持し、セレクターが安定して、オファーが期待どおりに表示および編集可能になりました。 （TGT-53434）

**[!UICONTROL Visual Experience Composer (VEC)]**

+++詳細を表示
* **[!UICONTROL Automated Personalization] （AP）または [!UICONTROL Multivariate Test] （MVT）アクティビティ内の場所の名前変更が、[!UICONTROL Targeting] の手順に移動して戻った後、保持されなかった。** 顧客は場所の名前を正常に編集および保存できるようになり、変更はアクティビティ作成プロセス全体を通して表示されたままになります。 （TGT-52367）
* **[!UICONTROL Stage] 環境のテナントに表示されるレガシー VEC UI**:[!UICONTROL Stage] 環境の特定のテナントにアクセスする際に、更新された VEC ではなく、レガシー UI が誤って表示される問題を修正しました。 この問題は複数のログインパスで再現可能で、[!UICONTROL Activities] の節の影響を受けました。 更新された UI が、[!UICONTROL Stage] 環境の両方のテナントで正しく表示されるようになりました。 （TGT-52261）
* **更新された VEC で背景色を選択すると、ページがクラッシュする**:
更新された VEC の [!UICONTROL Style] パネルから背景色を選択すると、ページがクラッシュして空白になる問題を修正しました。 コンソールエラーは、特に `querySelector` に関連する、未定義の要素からプロパティを読み取れなかったことを示しました。 クラッシュを引き起こすことなく、背景色を安全に選択できるようになりました。 （TGT-53561）
* **無効なユーザー入力エラーにより、アクティビティを保存できませんでした**：更新された VEC 内の既存のアクティビティを保存しようとした際に発生していたエラーを修正しました。 このエラーは編集時にのみ表示され、同じプロパティを持つ新しいアクティビティを作成する際には表示されませんでした。 エラーメッセージには次の内容が含まれます。
