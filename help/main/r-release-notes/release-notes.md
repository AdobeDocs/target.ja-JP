# [!DNL Target] リリースノート（最新）

[!DNL Adobe Target] の最新の機能、機能強化および修正点について説明します。 これらのリリースノートでは、[!DNL Target] API、SDK、[!DNL Adobe Experience Platform Web SDK]、at.js、その他のプラットフォームコンポーネントのアップデート（該当する場合）についても説明します。

（括弧内の問題番号は [!DNL Adobe] 内部で使用するためのものです。）

## [!DNL Target Standard/Premium] 25.10.1 （2025 年 10 月 22 日（PT））

このリリースには、次の更新および修正が含まれています。

**アクティビティ**

+++ 詳細を表示
* **更新された UI の操作性の問題を解決**。 レガシー UI と同様に、[!UICONTROL Observers] オプションを使用してアクティビティをプレビューで [!UICONTROL View Activity] るようになりました。 （TGT-51741）
* **[!UICONTROL Observer]ユーザーは、更新された UI でアクティビティのコンテンツを表示できるようになりました。** 更新されたアクティビティ UI で、監視者の役割のユーザーの表示が復元されました。 以前は、オブザーバーは、レガシー UI で使用できる機能である、変更、オファー、コンテンツの変更を表示できませんでした。 （TGT-53785）
* **[!UICONTROL Approver]ユーザーは、エディター権限エラーなしでアクティビティの目標を編集できるようになりました。** アクティビティ作成 UI の権限の問題で、承認者レベルのユーザーが高度な目標設定に対する変更を保存できなかった問題を修正しました。 影響を受けるユーザーには、十分なアクセス権があるにもかかわらず、エディター権限が必要な `403 Forbidden.Resource` エラーが発生しました。 （TGT-53819）

+++

**オーディエンス**

+++詳細を表示
* **複数オーディエンスの選択が「このアクティビティのみ」レポートで復元されました。** アクティビティの作成 UI の [!UICONTROL This activity only] の「オーディエンス」セクションで、ユーザーが複数のオーディ [!UICONTROL Goals & Settings] ンスを選択できない問題を修正しました。 （TGT-53283）
* **オーディエンスベースのレポートグラフにコンバージョンデータが正しく表示されるようになりました。** 「[!UICONTROL Reports]」タブで、デフォルト以外のオーディエンスを選択するとグラフが失敗する問題を修正しました。 データと信頼性の指標は使用できますが、ビジュアルグラフには実線のみが表示され、分析が困難になりました。 （TGT-53769）
* **[!UICONTROL Targeting] UI に、除外されたオーディエンスルールが明確に表示されるようになりました。** アクティビティの作成 UI の「[!UICONTROL Targeting]」セクションで、[!UICONTROL Exclude] に設定されたオーディエンスルールが明確に表示されない問題を解決しました。 これにより、特に、特定の URL を除くオーディエンスのターゲティングロジックを確認する際に混乱が生じました。 （TGT-53809）
* **オーディエンス定義の値が、「[!UICONTROL Targeting]」タブで選択可能かつコピー可能になりました。** アクティビティの作成インターフェイスで、「[!UICONTROL Targeting]」タブのオーディエンスルール値を選択およびコピーできなかった問題を修正しました。 この機能はレガシー UI では使用できましたが、更新された UI には含まれていません。 （TGT-53856）

+++

**ローカリゼーション**

+++詳細を表示
* **zh_CN ページエディターコンテキストでの「引用符」の誤訳を修正しました。** zh_CN ロケールのコンテキスト翻訳エラーを修正しました。「quote」という用語が「报价」として誤って翻訳されており、商業的な価格の見積もりを意味していました。 タイポグラフィ/見出しスタイル/引用ブロック セクションでは、意図した意味は価格設定ではなく、フォーマット要素、引用ブロックを指します。 （TGT-53841）
* **zh_CN ページエディターコンテキストでの「引用符削除」の誤訳を修正しました。** zh_CN ロケールの翻訳エラーを修正しました。「見積もり削除」が「移除了报价」として誤ってレンダリングされていました。これは、商業的な価格の見積もりを意味します。 タイポグラフィ/見出しスタイル/引用ブロック セクションでは、この用語は価格設定ではなく、書式設定要素、つまり引用ブロックを指します。 （TGT-53843）
* **zh_CN ページエディターコンテキストでの「引用符の並べ替え」の誤訳を修正しました。** zh_CN ロケールのコンテキスト翻訳エラーを修正しました。「見積りの並べ替え」が「重新排列了报价」として誤って翻訳されており、商業的な価格の見積もりを意味していました。 タイポグラフィ/見出しスタイル/引用ブロック セクションでは、この用語は価格設定ではなく、書式設定要素、つまり引用ブロックを指します。 （TGT-53844）
* **zh_CN ページエディターコンテキストでの「quote changed」の誤訳を修正しました。** zh_CN ロケールの翻訳エラーを修正しました。「quote changed」が「更改了报价」として誤ってレンダリングされていたので、商業的な価格の見積もりを示唆していました。 タイポグラフィ/見出しスタイル/引用ブロック セクションでは、この用語は価格設定ではなく、書式設定要素、つまり引用ブロックを指します。 （TGT-53845）

+++

**Recommendations**

+++詳細を表示
* **レコメンデーションに対する CSS セレクターの変更が正しく保存されるようになりました。** アクティビティの作成 UI で、ユーザーがレコメンデーション配置の CSS セレクターを更新できない問題を修正しました。 保存後に変更が元に戻され、ターゲティングコンテナの更新がブロックされる。 （TGT-53835）
* **レコメンデーションの変更でページ読み込みイベントの選択が保持されるようになりました。** レコメンデーションのイベントタイプを [!UICONTROL View] から [!UICONTROL Page Load] に切り替えた際に、ユーザーが変更を保存できなかったアクティビティの作成 UI の問題を修正しました。 選択は成功しているように見えましたが、移動してから元に戻り、アクティビティの公開をブロックしました。 （TGT-53957）

+++

**レポート**

+++詳細を表示
* **「[!UICONTROL Export order details to CSV]」で完全なデータがダウンロードされるようになりました。** 有効なレポートデータが存在する場合でも、「[!UICONTROL Overview]」オプションが空のファイルをダウンロードする問題を、更新された [!UICONTROL Export Order details to CSV] UI で解決しました。 （TGT-53787）

+++

**セキュリティ**

+++詳細を表示
* **IMS プレフィルターが追加され、GQL エンドポイントを組織間のデータ漏洩から保護できるようになりました。** 管理者タブのセキュリティの脆弱性が licenseGroups エンドポイントと targetProperties GraphQL エンドポイントに影響を与える問題を解決しました。 この問題は、JIL API を管理者クライアントトークンと共に使用することにより、組織間の製品データにアクセスするために不正利用される可能性があることから発生しました。 （TGT-53837）

+++

**Visual Experience Composer（VEC）**

+++詳細を表示
* **アクティビティ作成 UI でオーサリングの安定性が復元されました。** VEC UI で断続的に発生していたオーサリングに失敗し、リンクが予期せずクリック可能になり、ユーザーがページからリダイレクトされる問題を修正しました。 （TGT-53153）
* **アクティビティ作成 UI で保存されたアクティビティの編集が復元されました。** 変更を保存した後、ユーザーがアクティビティを編集できない問題を修正しました。 影響を受けたアクティビティは、「[!UICONTROL Applying initial modifications]」内に動かなくなり、それ以上の更新をブロックし、「[!UICONTROL Cancel]」ボタンを非表示にしました。 （TGT-53631）
* **VEC が「[!UICONTROL Applying initial modifications]」で停止しなくなりました。」** 多数の変更を含むエクスペリエンスを読み込む際に長い遅延が発生する VEC のパフォーマンスの問題を修正しました。 影響を受けるユーザーには、特にエクスペリエンス B シナリオで、UI が数分間「[!UICONTROL Applying initial modifications]」で停止していました。 （TGT-53727）
* **VEC は、ルート要素なしで変更を読み込むようになりました。**
VEC で、明確なルート要素を欠いた変更を読み込む際にエクスペリエンスが停止する問題を修正しました。 これらの変更により、UI が「A[!UICONTROL pplying initial modifications]」で無期限にハングしていました。 （TGT-53799）
* **アクティビティへの変更の保存が期待どおりに動作するようになりました。** 新しい作成 UI の権限関連の問題を解決し、アクティビティで目標と詳細設定を編集する際にユーザーが変更を保存できませんでした。 該当するユーザーには、適切なアクセス権があるにもかかわらず、赤いエラーリボンと「Forbidden.Resource」メッセージが表示されていました。 （TGT-53816）
* **VEC UI で、ビューをまたいでエクスペリエンスの変更を保持できるようになりました。** は、エクスペリエンス開発に影響を与えた更新された VEC の複数の問題を解決しました。 特にHTML オファーを使用する場合やビューを切り替える場合、変更が正しく保持されませんでした。 （TGT-53825）
* **変更が複数のエクスペリエンスにまたがる場合、すべてのビューが正しく表示されるようになりました。** アクティビティの作成 UI で、複数のビューに変更が適用された場合に 1 つのビューのみが表示される問題を修正しました。 変更が正しく適用されているにもかかわらず、ホバーツールチップに関連付けられたすべてのビューのリストが表示されませんでした。 （TGT-53827）
* **VEC が「[!UICONTROL Applying initial modifications]」で断続的に停止しなくなりました。** VEC でエクスペリエンスの読み込みに失敗して「[!UICONTROL Applying initial modifications]」でスタックしたままになる断続的な問題を修正しました。 この動作には一貫性がなく、リダイレクトループが発生する場合や、手動でのキャッシュのクリアが必要な場合がありました。 （TGT-53916）
* **VEC 読み込みの問題を再現できませんでした。** 既存のアクティビティを編集しても VEC が「[!UICONTROL Applying initial modifications]」で停止したままになる問題が報告されたことを調査しました。 このビヘイビアーは、親要素のないHTML コンテンツに関連していると疑われました。 アドビでは、繰り返し監視を継続し、安定性を確保するために、HTML オファー用の構造化コンテナを使用することをお勧めします。 （TGT-53972）
* VEC の **[!UICONTROL Design]モードが、断続的に [!UICONTROL Browse] モードのように動作しなくなりました。** UI で [!UICONTROL Design] モードが [!UICONTROL Browse] モードのように断続的に動作し、クリック可能な `<a>` リンクが許可され、要素を選択できなくなる問題を修正しました。 これにより、ホバーボックスが表示されなくなり、変更ワークフローがブロックされました。 （TGT-53136）
* **[!UICONTROL Composer] モードでボタンをクリックしても、ページリダイレクトはトリガーされなくなりました。** 更新された VEC UI で、[!UICONTROL Composer] モードでボタンをクリックすると、ボタンのターゲット URL への予期しないリダイレクトが発生する問題を解決しました。 このため、ユーザーがcall-to-action（CTA）要素を編集できず、オーサリングワークフローが中断されました。 （TGT-53137）
* **Automated Personalization アクティビティのオファーコードのアップデートが、エラーなしで保存されるようになりました。** 新しい作成 UI で、[!UICONTROL Invalid user input] アクティビティのオファーコードを更新する際に「[!UICONTROL Automated Personalization]」エラーが発生する問題を修正しました。 このエラーにより、入力が有効な場合でも、ユーザーは変更を保存できません。 （TGT-53586）
* VEC の **[!UICONTROL Design]モードで、編集可能なコンポーネントのリンクナビゲーションがブロックされるようになりました。** 更新された VEC で、[!UICONTROL Design] モードでもクリック可能な要素（ボタンやリンクなど）によってページリダイレクトがトリガーされる問題を解決しました。 この動作は [!UICONTROL Browse] モードに似ており、ユーザーは主要なコンポーネントを変更できません。 （TGT-53696）
* **「[!UICONTROL Clicked an element]」指標が、リダイレクトやエラーなしで機能するようになりました。** 新しい作成 UI で、「[!UICONTROL Clicked an element]」コンバージョン指標を選択する際に予期しないリダイレクトや空白の画面が発生する問題を修正しました。 要素を登録する代わりにセットアップがトリガーしたナビゲーション中にボタンをクリックすると、「[!UICONTROL element not found]」エラーが発生する。 （TGT-53817）
* **編集中に既存のアクティビティが無限の読み込みループに陥ることがなくなりました。新しい作成 UI**、VEC の既存のアクティビティを編集すると、ページが無限の読み込みループで停止したままになる問題を修正しました。 この問題は、新しく作成されたアクティビティに影響を与えず、ページ上の既存の変更によってトリガーされました。 （TGT-53913）
* **変更を含む既存のアクティビティページが VEC に正しく読み込まれるようになりました。** 更新された VEC で、保存された変更を含む既存のアクティビティを編集する際に、ページが読み込みフェーズでスタックされたままになる問題を修正しました。 新しいアクティビティは問題なく読み込まれましたが、以前に設定したアクティビティはレンダリングに失敗しました。 （TGT-53967）
