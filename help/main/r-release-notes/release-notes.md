# [!DNL Target] リリースノート（最新）

[!DNL Adobe Target] の最新の機能、機能強化および修正点について説明します。 これらのリリースノートでは、[!DNL Target] API、SDK、[!DNL Adobe Experience Platform Web SDK]、at.js、その他のプラットフォームコンポーネントのアップデート（該当する場合）についても説明します。

## [!DNL Target Standard/Premium] 26.1.2 （2026 年 1 月 30 日（Pt））

**Adobe Target Insights ダッシュボード**

Adobe Targetには、組織が Target をどのように使用しているかの概要を表示する、新しいインサイトダッシュボードが含まれるようになりました。 ダッシュボードには、ライブアクティビティ、ライブおよび変更済みアクティビティ、完了したアクティビティ、公開済みアクティビティ、A/B テストアクティビティなどの主要指標が表示されます。 時間範囲セレクターを使用して、2025 年の振り返りビューを含む、様々な期間のトレンドを調べます。 このダッシュボードは、継続的な採用状況とアクティビティの推移の追跡方法として引き続き使用できます。

[Adobe Target Insights ダッシュボードを参照してください ](/help/main/c-activities/insights-dashboard.md)

## [!DNL Target Standard/Premium] 26.1.1 （2026 年 1 月 18 日（PT））

**アクティビティ**

+++詳細を表示

* **アクティビティをコピーできません – ユーザー入力が無効です。** アクティビティをコピーする際に、ユーザーに参考にならない「無効なユーザー入力」エラーが表示される問題を修正しました。 以前は、アクティビティを複製すると、そのワークスペース固有のプロパティの割り当てが保持されず、ABActivity がデフォルト以外のワークスペースに属するプロパティを少なくとも 1 つ必要とするので、バックエンドが保存リクエストを拒否していました。 この不一致により、UI で一般的なエラーが発生し、ユーザーにガイダンスが提供されませんでした。 この修正により、コピー操作中にワークスペースの割り当てが正しく保持され、ユーザーが変更を加えることなくコピーされたアクティビティを保存できるようになり、誤解を招く検証エラーを防ぐことができます。 （TGT-54282）
* **Web エディターが提供するワークスペース列を有効にします。** この更新は、web エディター内の他のワークスペースに表示される [!UICONTROL Default Workspace] からのオファーによって発生する顧客の混乱に対処します。 この動作は設計通りに動作しますが、すべてのワークスペースで意図的にオファーが表示される [!UICONTROL Default Workspace]、顧客からは、特に「承認者」などのデフォルト以外のワークスペースでアクティビティを作成する場合に、UI によってワークスペースオリジンが明確にされなかったという報告がありました。 わかりやすくするために、web エディターのオファーリストで「[!UICONTROL Workspace]」列が有効になりました。これにより、各オファーが属するワークスペースをユーザーが簡単に区別でき、表示される追加のオファーの誤った解釈を防ぐことができます。 （TGT-54138）
* **target=&quot;_blank&quot;を使用したリンク」が新しいタブで開きます。** この修正により、「target=&quot;_blank~~ を含む作成済み web サイトの [!UICONTROL Browse] モードでクリックすると、新しいブラウザータブで開き、エディター内のプレビューエクスペリエンスが中断される問題が解決されます。 この動作が発生したのは、アンカー要素が変換され、エディター内でナビゲーションを維持するためにターゲットが上書きされた従来の UI とは異なり、作成したページのネイティブリンク属性が、拡張機能が挿入したJavaScriptによってインターセプトされないからです。 この更新により、「target=&quot;_blank~~ を使用するリンクが web エディターで適切に処理されるようになり、オーサリング中に外部タブが開かなくなりました。 （TGT-54134）
* **プロパティ警告を選択解除。** この更新では、アクティビティエディターで自動検出されたプロパティの選択を解除する際に、ユーザーにわかりやすく通知する視覚的な警告が導入されました。 以前は、自動検出されたプロパティを削除しても、そのプロパティが永続的に削除されるとは示されていませんでしたが、この結果、ターゲティング設定が誤って失われる可能性がありました。 この修正により、レガシー UI の動作と一致する警告アイコンが追加され、プロパティの選択を解除するとアクティビティから削除されることをユーザーに通知します。 （TGT-54121）
* 「**[!UICONTROL Workspaces]」セクショ [!UICONTROL Users] のドロップダウンは 20 に制限されます。** この修正により、ユーザーが多数のワークスペースにアクセスした場合でも、[!UICONTROL Workspaces] / [!UICONTROL Administration] セクションの [!UICONTROL Users] ドロップダウンに 20 個のワークスペースしか表示されない問題が解決されました。 また、`licenseGroups` に対する基になるGraphQL呼び出しも 20 件の結果に制限されているので、組織内のより多くのワークスペースにアクセスできるにもかかわらず、UI に不完全なリストが表示されていました。 更新によってこのハードリミットが削除されるので、使用可能なワークスペースの完全なセットが返され、正しく表示されるようになりました。 （TGT-53820）
* **オファーモーダルにワークスペース列が表示されない問題を修正しました。** オファーモーダルで、更新された UI にワークスペース列が表示されない問題を修正しました。 これにより、[!UICONTROL Default Workspace] ージのオファーが、選択したワークスペースのオファーと並んで表示され、接触チャネルの兆候が見られず、顧客の混乱を招きました。 ワークスペース列が有効になり、顧客は各オファーが属するワークスペースを明確に識別できるようになりました。 （TGT-52320）

**プロパティ**

+++詳細を表示

* **既に削除されている場合、アクティビティの編集で、自動検出されたプロパティを追加しないでください。** この修正は、アクティビティを編集すると、以前にユーザーが削除した、自動検出されたプロパティが自動的に再導入される問題を修正します。 編集するアクティビティを再度開くと、削除されたプロパティがシステムによって誤って復元され、[!UICONTROL Properties List] ークフローでの動作に一貫性がなくなり混乱が生じました。 このアップデートにより、自動検出されたプロパティが削除されると、その後のすべての編集中に削除されたままになり、ユーザーが明示的に追加し直さない限り、再び表示されなくなります。 （TGT-54182）
* **自動検出されたプロパティは、既に削除されている場合は追加しないでください。この修正により、自動検出されたプロパティをアクティビティからユーザーが手動で削除した**、アクティビティエディター内での後続のナビゲーション中にシステムによって再導入されることがなくなります。 以前は、ユーザーが自動検出プロパティの選択を解除して [!UICONTROL Targeting] ステップに移動し、[!UICONTROL Experiences] に戻ると、アクティビティエディターの状態スライスに格納されている自動検出リストに基づいて、削除されたプロパティが再入力されていました。 更新されたロジックでは、自動検出されたプロパティと ~ActivityState~ スライス内の現在のプロパティが比較されるようになり、ユーザーが既に削除した、自動検出されたプロパティの再追加を防ぎます。 これにより、ステップ間で一貫した動作が行われ、ユーザーの意図に従います。 （TGT-54181）
* **自動検出されたテキストをプロパティリストに追加します。** この機能強化により、システムによって自動的に検出されたプロパティに明確なラベルを付けるために [!UICONTROL Properties List] が更新されます。 ユーザーに表示される [!UICONTROL Properties List] に自動検出されたプロパティも存在する場合、「ActivityEditorSlice~ 状態に保存された値を使用して、その名前の横に「（自動検出 ~」テキストが表示されるようになりました。 これにより、従来の UI の動作が反映され、ユーザーは手動で選択したプロパティと、自動的に識別されるプロパティを簡単に区別できます。 （TGT-54120）
* **自動検出 [!UICONTROL Properties] を状態に追加します。この更新により、** ActivityEditorSlice.ExperienceEditor~ 状態 ~、Web エディターから「Activity [!UICONTROL Experiences]」タブに渡されるすべての自動検出プロパティ ID の最新リストを一貫して維持できるようになります。 ユーザーが「[!UICONTROL Experiences]」タブに移動するたびに、新しく検出されたプロパティで状態が更新されるので、重複を防ぎ、正確なトラッキングと信頼性の高いダウンストリーム動作を実現できます。 （TGT-54119）

**レコメンデーション**

+++詳細を表示

* ドロップダウ **[!UICONTROL Environment]に表示される結果は 100 件のみです。** この修正は、環境が 100 を超えるお客様の場合、[!UICONTROL Environment] 内の [!UICONTROL Recommendations] ドロップダウンに最初の 100 個のエントリしか表示できないという制限に対応しています。 基になるGraphQL クエリ（~getEnvironmentsV2~）は、ハードコーディングされたページサイズ 100 でページ分割されていました。そのため、他のページが使用可能な場合でも、UI には部分的なリストのみが表示されます。 環境が 100 以上あるお客様の場合、この問題が原因でオプションが見つからず、選択エクスペリエンスが不完全でした。 この更新により制限が増えるので、すべての環境が返されて表示され、環境数に関係なく完全な可視性が確保されます。 （TGT-53903）

**レポート**

+++詳細を表示

* **[!UICONTROL Reports] 矢印が展開可能な列を明確に示さない問題を修正しました。** 更新された UI で、追加の列を展開できることをレポートテーブルが明確に表示しない問題を修正しました。 列ヘッダーの近くの [!UICONTROL Reports] 矢印に、より多くの列が使用可能であることを顧客が理解するのに役立つ、表示されないツールチップが追加されました。

**ビュー**

+++詳細を表示

* **ビューに適用された変更を削除できない。** この修正により、変更を最初に追加のビューに再適用しない限り、ユーザーがアクティビティ内の変更を削除できなかった問題が解決されます。 アクティビティ（アクティビティ ID 302467 など）を編集する際、変更を削除しようとしても効果がなく、ユーザーは不要な変更を削除できませんでした。 ただし、「その他のビューに適用」を使用して変更を再適用し、`Page Load` イベントに割り当てると、削除は突然期待どおりに動作しました。 （TGT-54088）

**[!UICONTROL Visual Experience Composer]（VEC）**

+++詳細を表示

* 新 **[!UICONTROL Experience Fragment]い VEC UI で名前が切り詰められました** （TGT-54312）
* **指標に [!UICONTROL Advanced Settings] を使用でき [!UICONTROL Revenue] せん。** この修正では、[!UICONTROL Advanced Settings] で [!UICONTROL Revenue] 指標の [!UICONTROL Goals & Settings] を設定する際に、403 「アクセスが拒否されました」というエラーが発生する問題を修正しました。 この問題は、主な目標に関連付けられた依存関係条件を追加する際に発生しました。アクティビティの作成と編集に十分な権限を既に持っているユーザーであっても、バックエンドではエディター権限が誤って必要でした。 その結果、有効な設定にもかかわらず、アクティビティの保存に失敗しました。 この更新では権限チェックが修正されるので、適切なアクセス権を持つユーザーは、禁止リソース エラーをトリガーすることなく、売上高指標の依存関係を正常に追加できます。 （TGT-54092）
* **選択した画像に「追加」ボタンが適用されない問題を修正しました。** アクティビティ作成プロセスで画像を選択または更新する際に、お客様が特定の画像を追加できなかった問題を修正しました。 顧客が特定のアセット（例えば、「ipp」の検索時に返された画像）を検索したとき、「[!UICONTROL Add]」ボタンをクリックしても、選択した画像が適用されず、変更は作成されませんでした。 `Homepage-banner-1-moz.jpg` などの他の画像の選択は、引き続き期待どおりに動作しました。 この更新により、更新された UI で、有効なすべての画像を一貫して適用できるようになります。 （TGT-53610）
* **URL 条件を削除すると、目標指標の設定がリセットされる問題を修正しました。** [!UICONTROL Goal] 指標で単一の URL 条件を削除すると、更新された UI で設定全体がリセットされる問題を修正しました。 お客様が [!UICONTROL Conversion]/[!UICONTROL Viewed a Page] で保存済みの URL 条件を削除しようとすると、目標タイプが予期せず [!UICONTROL Viewed an Mbox] に切り替わり、以前に設定したすべての設定が削除されました。 この更新により、選択した URL 条件のみが削除され、残りのすべての目標設定はそのまま維持されます。 （TGT-53271）
* **検索でサブフォルダーを調べない問題を修正しました。** オファーを検索しても、更新された UI のサブフォルダーから結果が返されない問題を修正しました。 オファーを見つけるには、オファーが保存されているフォルダーに手動で移動する必要があり、検索動作が API 機能と一致しなくなりました。 検索ではフォルダー内を再帰的に検索できるようになりました。これにより、顧客は各フォルダーを個別に開かなくてもオファーを見つけることができます。 （TGT-51954）
