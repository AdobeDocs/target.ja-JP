# フラグの概要 {#introduction}

>[!AVAILABILITY]
>
>フラグは現在Betaにあり、Adobe Targetのお客様が利用できます。 Adobeの担当者に連絡してアクセス権をリクエストしてください。

Adobe Targetのフラグは、機能をプロダクションに展開しながら、誰がいつ見るかを正確に制御できる制御リリースシステムです。 機能は本番環境にあり、エンドユーザーには表示されない可能性があります。その後、再展開することなく、ターゲットオーディエンスに応じて段階的にオンにすることができます。

## 開発から制作まで {#dev-to-prod}

フラグは、最初の開発段階から一般提供までの機能のジャーニー全体をサポートします。

1. **開発者テスト** – 開発者が機能フラグを作成し、そのコードを任意の環境にデプロイし、独自のセッションのみに対してテストを実行します。 他のユーザーは影響を受けず、分岐は必要ありません。

2. **ターゲットプレビュー** – 製品オーナーは、オーディエンスを機能フラグにリンクし、その機能を定義されたユーザーのサブセット（ベータ版の参加者や特定の地域など）が検証やフィードバックのために利用できるようにします。

3. **段階的なロールアウト** – この機能は、1%、10%、50%、100%、より多くのユーザーに段階的に開かれ、任意の手順で機能を一時停止、調整、またはオフにすることができます。

4. **一般提供** – 検証が完了すると、この機能はすべてのユーザーが利用できるようになります。

## 主な能力 {#capabilities}

フラグは、次のような機能を提供するCMSです。

* **機能フラグ** — コードを再デプロイすることなく、ターゲットオーディエンスの実行時に任意の機能をオンまたはオフにします。

* **オーディエンスターゲティング** — コンテキスト属性を使用して機能を表示するユーザーを制御します。

* **機能グループ** – 複数の関連する機能フラグをアプリケーション間でバンドルし、それらを単一のユニットとして管理することで、同じオーディエンスに一貫性のあるエクスペリエンスを提供します。

* **段階的なロールアウト** — リスクを軽減し、フィードバックを収集し、バックエンドの負荷を管理するために、機能の配信を段階的に進めます。

* **Kill switch** — コードの変更や再デプロイメントを行うことなく、問題が検出された場合は、すぐに任意の機能をオフにします。

* **AEP SDK サポート** — フラグはAEP Web SDKおよびAEP モバイル SDKを介してデプロイされ、web アプリケーションとモバイルアプリケーション間で一貫したフラグ評価が可能になります。
