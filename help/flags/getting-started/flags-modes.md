# フラグモード {#modes}

フラグには、2つの異なる機能ターゲットモードが用意されています。 それぞれ、異なるリリース管理の目的を兼ね備え、特定のタイプのアプリケーションやユースケース向けに設計されています。

>[!TIP]
>
>個々のユーザーセッションまたは組織と環境の範囲に基づいて、モードを選択します。

## ユーザーレベルのターゲティング {#user-level}

### 概要 {#user-level-overview}

ユーザーレベルのターゲティングにより、個々のユーザーレベルでの機能のロールアウトが可能になります。 特定のユーザー、社内テスター、カナリールユーザー、パーセントベースの段階的なロールアウトなどの正確なターゲティングをサポートしています。

このモードは、ログインしているユーザーに基づいてエクスペリエンスを変更する必要があるアプリケーションに最適です。

### 使用するタイミング {#user-level-when}

次のような場合に、ユーザーレベルのターゲティングを使用します。

* 特定ユーザーに対する機能の有効化
* A/B テストの実施
* 小規模なグループでカナリテストを実施してから、より広範に展開する
* 機能をパーセンテージで徐々に展開する
* 個々のユーザー属性（メールドメインやプロファイルデータなど）にもとづいて、エクスペリエンスを微調整します

### 制限事項 {#user-level-limitations}

ユーザーレベルのターゲティングは、次のシナリオ向けに設計されていません。

* 組織、環境、または地域レベルで機能を制御しない
* テナント全体の機能の有効化またはエンタイトルメントのゲート化を目的としていない

## 組織および環境レベルのターゲティング {#org-level}

### 概要 {#org-level-overview}

組織レベルのターゲティングにより、組織、環境、地域スコープでの機能のロールアウトが可能になります。 個々のユーザーではなく、組織全体または特定のデプロイメント環境での機能の可用性を制御します。

このモードは、エンタープライズレベルの機能ゲーティングおよび環境固有のロールアウト向けに設計されています。

### 使用するタイミング {#org-level-when}

組織レベルのターゲティングは、次のような場合に使用します。

* 組織全体で機能を有効にする
* 環境（ステージと実稼動など）別の機能の可用性の管理
* 地域ロールアウトの実行
* エンタイトルメントまたはサブスクリプション階層に基づくゲートフィーチャー

### 制限事項 {#org-level-limitations}

組織および環境レベルのターゲティングは、次のシナリオ向けに設計されていません。

* リッチユーザープロファイルベースのターゲティングをサポートしていない
* 個人ユーザーレベルでのパーセンテーションベースのロールアウトはサポートしていません
* 実験やA/B テストを目的としていない

## 比較 {#comparison}

| ディメンション | ユーザーレベルのターゲティング | 組織および環境レベルのターゲティング |
|---|---|---|
| 制御範囲 | 個人ユーザー | 組織、環境、地域 |
| ターゲティング基準 | ユーザープロファイルと属性 | 組織と環境のコンテキスト |
| パーセンテージ ロールアウト | ○ | × |
| A/Bテスト | ○ | × |
| エンタープライズゲート | × | ○ |

## 適切なモードの選択 {#choosing}

* 質問が&#x200B;*の場合「この機能を表示するユーザーはどれですか？」* → **ユーザーレベルのターゲティング**&#x200B;を使用
* 質問が&#x200B;*の場合、「この機能を備えるべき組織または環境はどれですか？」* → **組織および環境レベルのターゲティング**&#x200B;を使用
