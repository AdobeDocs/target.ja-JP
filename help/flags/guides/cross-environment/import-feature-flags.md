# 機能フラグの読み込み {#import-feature-flags}

フラグを使用すると、1つのサンドボックス（サンドボックス 1など）から別のサンドボックス（サンドボックス 2など）にフィーチャーフラグを読み込むことができます。 これにより、フラグ設定を手動で再作成する必要がなくなり、サンドボックス間の設定ドリフトのリスクを軽減できます。

## 手順1：宛先サンドボックスとアプリケーションに移動する {#step-1}

**destination** サンドボックス （フラグを読み込むサンドボックス）のコンソールにログインします。 「**機能フラグの読み込み**」をクリックし、アプリケーションドロップダウンからフラグを読み込むアプリケーションを選択します。

![機能フラグページの「機能フラグを読み込む」ボタン &#x200B;](assets/import-feature-flags.png)

>[!IMPORTANT]
>
>現在のサンドボックスと選択したアプリケーションは、ソースではなく&#x200B;**宛先**&#x200B;である必要があります。 例えば、サンドボックス 1からサンドボックス 2にフラグを読み込むには、サンドボックス 2 コンソールにログインして、サンドボックス 2 アプリケーションを選択します。

## 手順2：読み込みダイアログを開く {#step-2}

**機能フラグの読み込み**&#x200B;を選択します。 利用可能なアプリケーションに基づいて事前に入力された、ソースサンドボックスとアプリケーションを示すダイアログが開きます。 必要に応じて、ダイアログのドロップダウンからソースサンドボックスとアプリケーションを変更できます。

## 手順3：読み込む機能フラグの選択 {#step-3}

ソースサンドボックスの機能フラグのリストから、読み込むフラグを選択します。 一度に1つ、複数、またはすべてのフラグを選択できます。

## 手順4：読み込むフィーチャーフラグの状態を選択する {#step-4}

ドロップダウンを使用して、機能フラグの読み込み方法を選択します（**有効**、**無効**、または&#x200B;**現在の状態**）。 デフォルトでは、機能フラグは&#x200B;**無効**&#x200B;状態で読み込まれます。

## 重要なメモ {#important-notes}

機能フラグを読み込む際は、次の点に注意してください。

* 同じキーを持つ機能フラグが宛先サンドボックスに既に存在する場合、そのフラグは読み込まれません。
