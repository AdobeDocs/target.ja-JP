# アプリにフラグを統合 {#integrate}

このセクションでは、アプリケーションタイプ別に整理された、すべてのフラグクライアントの統合ガイドラインを提供します。

## 統合パスの選択 {#choose}

アプリケーションの種類に合ったガイドを選択します。

1. [&#x200B; スタートアップガイド &#x200B;](startup-guide.md) – すべての統合手順の概要については、ここから始めてください
2. [&#x200B; デスクトップアプリケーション &#x200B;](desktop-applications.md) — デスクトップアプリケーション用のSDK統合
3. [&#x200B; モバイルアプリ &#x200B;](mobile-applications.md) — モバイルアプリ用SDK統合
4. [Web アプリケーション &#x200B;](web-applications.md) — web プロパティ用SDK統合
5. [Web サービス &#x200B;](web-services.md) — バックエンドサービス用のサーバーサイド SDK統合
6. [SDK](sdks.md) — SDK アーキテクチャ、前提条件、使用可能なSDK
7. [統合手順](integration-steps.md) – 詳細なステップバイステップ統合手順

## アプリケーションタイプの概要 {#overview}

| アプリケーションタイプ | 連携の強化 |
|---|---|
| **Web サービス / バックエンド** | Java SDKまたはNode.js SDK |
| **Web アプリケーション** | Web SDK（近日リリース予定） |
| **モバイルアプリケーション** | モバイルSDK（近日リリース予定） |
| **デスクトップアプリケーション** | SDK（近日リリース予定） |
