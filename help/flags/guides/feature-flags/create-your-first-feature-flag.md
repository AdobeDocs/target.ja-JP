# 最初の機能フラグを作成 {#create-feature-flag}

## 前提条件 {#prerequisites}

機能フラグを作成する前に、次の手順を実行します。

* フラグコンソールにアクセスできます。[ コンソールへのログイン ](../console/log-in-to-the-console.md)を参照してください。
* アプリケーションがオンボーディングされました。[ アプリケーションのオンボーディング ](../applications/onboard-your-application.md)を参照してください。
* **製品リリース所有者**&#x200B;の役割があります

## 手順1：機能フラグの作成 {#create}

新しい機能フラグを作成するには、コンソールで次の手順に従います。

1. **フラグ コンソール**&#x200B;にログインし、左側のパネルに移動して、**機能フラグ**&#x200B;を選択します。
1. 「**アプリケーション**」ドロップダウンからアプリケーションを選択します。
1. **新機能フラグ**&#x200B;を選択します。
1. フォームフィールドに入力します。

   | フィールド | 説明 |
   | --- | --- |
   | **Name** | 機能フラグの表示ラベル。 コードでは使用されません。 |
   | **キー** * | コードでフラグを評価するために使用される識別子。 作成後に変更することはできません。 |
   | **説明** | ドキュメント用のオプションの説明。 |
   | **メタデータ** | オプション。 最大1,024文字。 フラグに関連付ける追加のメタデータには、このフィールドを使用します。 |
   | **タグ** | ドキュメント用のオプションのタグ。 |
   | **ID** * | フラグが評価されるID （ECIDなど）。 これは、機能リクエストで渡されたIDです。 |
   | **パーセンテージのロールアウト** | この機能を提供する定義済みオーディエンスの割合。 デフォルトは100%です。 [徐々にロールアウトする機能の設定](set-feature-gradual-rollout.md)を参照してください。 |

   * でマークされたフィールドは必須です。

>[!IMPORTANT]
>
>**キー**&#x200B;は、コードで使用されている識別子であり、作成後に変更することはできません。 キー&#x200B;**にはスペース**&#x200B;を含めることができず、**大文字と小文字を区別**&#x200B;します。 **名前**&#x200B;は表示ラベルのみで、コードでは使用されません。2つは独立しています（名前はキーに変換されません）。 「キー」フィールドにスペースを入力すると、次のエラーが発生します：_「機能キーの値が無効です」_

1. オプションでオーディエンス基準を追加します（手順2を参照）。
1. 機能フラグ設定を保存します。

## 手順2：オーディエンス条件の追加 {#audience}

オーディエンス基準は、どのユーザーに機能を表示するかを制御します。 **コンテキスト属性** – 機能リクエストでweb サイトまたはアプリが送信する値（`locale`または`platform`など）を持つユーザーをターゲットにできます。 **AND**、**OR**、**NOT**&#x200B;と組み合わせます。 [ オーディエンスルールでコンテキストを使用](../audience/using-context-in-audience-rules.md)を参照してください。

オーディエンス条件を追加するには、機能フラグを作成または編集する際に「**オーディエンス**」タブに移動します。
