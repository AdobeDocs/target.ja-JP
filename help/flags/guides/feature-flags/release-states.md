# リリース状態 {#release-states}

リリースマネージャーは、コンソールナビゲーションバーからリリースの状態を直接更新できます。 状態は、リリースが公開されているか、テストに限定されているか、完全にロールアウトされているか、クローズされているかを制御します。

## 使用可能な状態 {#states}

| # | 都道府県 | 説明 | 変更可能 |
|---|---|---|---|
| 1 | **Draft** | リリースはシステムに保存されますが、まだアクティブではありません。 オーディエンスルールは適用されません。 | すべての変更を許可（リリース、機能、オーディエンス）。 |
| 2 | **保存済み** | リリースが登録され、スロットが割り当てられます。 このリリースは、まだユーザーには表示されません。 | すべての変更を許可します。 |
| 3 | **公開済み** | リリースは、指定したオーディエンスに対して公開されます。 オーディエンスの基準を満たしたユーザーには、その機能が提供されます。 | すべての変更を許可します。 |
| 4 | **フル ロールアウト** | このリリースは、オーディエンスルールに関係なく、すべてのユーザーが利用できます。 すべてのオーディエンスルールが削除されます。 | 機能の変更は許可されています。 オーディエンスは変更できません。 |
| 5 | **ベースライン化** | すべてのユーザーには、このリリースの機能がデフォルトの動作として用意されました。 条件付きコードは削除できます。 リリーススロットが解放されます。 | 変更は許可されていません。 |
| 6 | **中止** | リリースが停止されました。 このリリースから機能を受け取るユーザーはありません。 リリーススロットが解放されます。 | 変更は許可されていません。 |

## 許可されたトランジション {#transitions}

すべての状態遷移が許可されているわけではありません。 許可されたパスを理解することで、ロールアウトを計画し、状態の変更を管理する際のエラーを回避できます。

* リリースは、次の状態で進めることができます。ドラフト→保存され→公開済み→ フルロールアウト → ベースライン
* リリースは、ドラフト、保存、公開、またはフルロールアウトから中止できます
* ベースライン化または中止されると、それ以上の変更は許可されません

## リリースのキャパシティ {#capacity}

アクティブな（保存済みまたは公開済み）リリースの数は、常に制限されています。 容量を解放するには：

* すべての参加アプリケーションがコンディショナルコードを削除したら、完了したリリースを&#x200B;**ベースライン化**&#x200B;に移動します。
* 失敗した&#x200B;**中止** リリースは続行されません。

3か月以内にリリースをベースライン化または中止することを計画している場合。
