# リリースオーディエンスルールの更新 {#update-release-audience-rules}

## オーディエンス設定へのアクセス {#access}

リリースを受け取るユーザーの設定を開始するには、コンソールのオーディエンス設定に移動します。

1. フラグコンソールでリリースを開きます。
2. 「**オーディエンス**」タブに移動します。
3. 「**オーディエンスルール**」の横にあるトグルを有効にします。
4. **包含ルール** （リリースを受け取る必要がある）または&#x200B;**除外ルール** （そうでない）を追加するかどうかを選択します。

## サポートされているオーディエンス条件 {#criteria}

### ID のタイプ {#id-type}

アカウントタイプ（個人アカウントやエンタープライズアカウントなど）にもとづいて、ユーザーをターゲティングします。

### 国 {#country}

登録国にもとづいてユーザーをターゲティングします。

### ユーザー ID {#user-id}

一意のユーザーID （GUID）で特定のユーザーをターゲティングします。

### 電子メール アドレス {#email}

正確なメールアドレスで利用者をターゲティングする： 複数のメールアドレスを一度に追加するには、演算子ドロップダウンから「**in**」を選択し（**is**）の代わりに）、次の操作を行います。

* テキストボックスにアドレスのコンマ区切りリストを入力するか、
* 行ごとに1つのアドレスを持つ`.txt` ファイルをアップロードするか
* アドレスのリストを含む`.csv` ファイルをアップロード

リストの規模が非常に大きい場合は、それを小さなバッチに分割します。

### メールドメイン {#email-domain}

電子メールアドレスが特定のドメインに属するすべてのユーザー（例：`yourcompany.com`）をターゲットにします。

### クライアント IP {#client-ip}

クライアントのIP アドレスに基づいてユーザーをターゲティングします。

### 割合（％） {#percentage}

未認証のユーザーを含め、すべてのユーザーのランダムな割合でターゲティングします。

### 割合（認証済みのみ） {#percentage-auth}

認証済みユーザーのランダムな割合のみをターゲットにします。 未認証のユーザーは除外されます。

## 複数のルールの組み合わせ {#combining-rules}

AND/OR ロジックを使用して、複数のオーディエンス条件を組み合わせることができます。

**例：**

* 特定のドメインのエンタープライズアカウントのみをターゲットにするには、ANDを使用して&#x200B;**ID type** ルールと&#x200B;**Email domain** ルールを組み合わせます。
* 特定の国の10%のユーザーをターゲットにするには、**Country** ルールと&#x200B;**Percentage** ルールを組み合わせます。

条件を追加または削除するには、**+/-** アイコンを使用します。 条件を複製するには、既存のルールの横にある複製アイコンを選択します。 複雑なネストされたロジックの場合は、**ネストされたロジック**&#x200B;を有効にし、テキストボックスに有効な式を入力します。
